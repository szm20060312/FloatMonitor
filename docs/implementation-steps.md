## 第 1 步：创建 Xcode 项目骨架

### 目标
在 Xcode 中创建项目，搭建基本目录结构。

### 具体任务
1. 使用 Xcode 26.5 创建新 macOS App 项目
   - Template: macOS → App
   - Interface: SwiftUI
   - Lifecycle: SwiftUI App
   - Language: Swift
2. 创建目录结构：
   ```
   cpu_mem_tool/
   ├── App/cpu_mem_toolApp.swift
   ├── MenuBar/MenuBarView.swift
   ├── Window/ContentView.swift
   ├── Services/SystemMonitorService.swift
   ├── Models/SystemStats.swift
   └── Utils/
   ```
3. 配置 Info.plist：
   - 可选 LSUIElement = YES（隐藏 Dock 图标）
4. 基础 App 入口代码

### 验证标准
- [ ] `swift build` 或 Xcode Build 通过
- [ ] 应用可启动，显示空白窗口
- [ ] 目录结构清晰，文件组织合理

## 第 2 步：系统监控数据采集层

### 目标
编写 SystemMonitorService 获取实时系统数据。

### 具体任务
1. 定义 `SystemStats` 数据模型
2. 实现 CPU 监控（`host_processor_info`）
3. 实现内存监控（`host_statistics64`）
4. 实现温度监控（IOKit SMC）
5. 实现网络监控（`sysctl`）
6. 实现 GPU 监控（IOKit `IOAccelerator`）
7. 使用 Timer + @Published 实现定时刷新
8. 打印日志确认数据采集正常

### 验证标准
- [ ] 控制台每秒输出一次完整系统数据
- [ ] CPU 数值合理（0-100%）
- [ ] 内存数值与活动监视器一致
- [ ] 温度、网络、GPU 数据采集无崩溃
